## Project Overview

paperwrite is a CLI scaffolding tool for structured academic paper writing ("vibe writing"). The core principle is "think before you write" — users progress through three layers: sparks (free thinking in Markdown) → outline (paragraph-level outlines in Markdown) → paper (final LaTeX). The tool provides `init`, `build`, and `status` commands.

## Build & Run Commands

```bash
# Install in development mode
pip install -e .

# Run CLI
paperwrite init <name>    # Create new paper project
paperwrite build          # Compile PDF (requires tectonic)
paperwrite status         # Show writing progress

# Build PDF via Makefile (uses conda env "paperwrite")
make pdf
make clean
make watch
```

There are no tests currently (`test/` is empty). There is no linting configuration.

## Architecture

The project has two distinct parts:

**1. The tool itself** (`cli/` package):
- `cli/cli.py` — Click-based CLI with three commands: `init`, `build`, `status`. Uses `_find_project_root()` to locate a paperwrite project by walking up from cwd looking for `paper/main.tex`.
- `cli/templates.py` — All template content as string constants. `DIRS` lists scaffold directories, `FILES` maps relative paths to template content, `TEMPLATE_SIGNATURES` is auto-generated for status detection (compares file content against templates to determine "not started" vs "written").
- `pyproject.toml` — Entry point is `cli.cli:main`.

**2. The paper project scaffold** (generated by `paperwrite init`):
- `sparks/` — 7 Markdown files for free-form thinking (motivation, innovations, experiments, results, related_work, writing_log, questions)
- `outline/` — 9 Markdown files for structured outlines (structure, abstract, introduction, related_work, method, experiments, results, discussion, conclusion)
- `paper/` — LaTeX source: `main.tex` pulls in `preamble.tex` and `paper/sections/*.tex`; `references.bib` for citations
- `refs/`, `scripts/` — for papers, notes, data, and figure/table scripts

The writing workflow is strictly upstream-first: sparks → outline → paper. Changes should always propagate from upstream to downstream.
